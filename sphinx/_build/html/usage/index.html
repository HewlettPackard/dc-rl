Usage

Reinforcement learning algorithms

DCRL-Green supports the two multi-agent training approaches: independent learning approach such as IPPO [3], IA2C that decompose the problem into single-agent tasks for learning and centralised learning approach such as MADDPG [5] that uses a centralised critic with decentralised execution.

The following command can be used to run the MARL algorithm of choice: (For execution using EnergyPlus model, launch the Sinergym environment before running the script (details can be found in EnergyPlus enabled DC simulation (DCRLeplus)))

Name

DCRL Implementation

DCRLeplus Implementation

IPPO

 
python train_ppo.py

EPLUS=1 python train_ppo.py

IA2C

 
python train_a2c.py

EPLUS=1 python train_a2c.py

MADDPG

 
python train_maddpg.py

EPLUS=1  python train_maddpg.py

Note

User configurations described in the following sections can be modified by directly editing the above python scripts.

Carbon Intensity data

This dataset includes hourly average carbon intensity (CI) information for a given location. DCRL-Green includes CI data for 3 locations (NY, AZ, WA) collected by US Energy Information Administration. Users can upload CI dataset of choice in the repository, under dc-rl/data/CarbonIntensity and use the variable 'cintensity_file' to specify the filename.

Note

CI dataset should be saved in .csv format and should contain CI data for one year (24*365), in a hourly way.

Example:

'cintensity_file': 'NYIS_NG_&_avgCI.csv'

The above default dataset consists CI data for a period of one year starting from 1/1/2022 to 12/31/2022.

Weather data

Description of weather dataset. What info does it give? The default weather data in DCRL-Green was obtained from the EnergyPlus [2] weather files collection. Users can upload weather dataset of choice in the repository, under dc-rl/data/Weather and use the variable 'weather_file' to specify the filename.

Note

Weather dataset should be saved in .epw format

Example:

'weather_file': 'USA_NY_New.York-Kennedy.epw'

Workload data

This dataset provides hourly IT workload information. The default weather data in DCRL-Green was obtained from the Alibaba open source database [1]. Users can upload IT workload dataset of choice in the repository, under dc-rl/data/Workload and use the variable "workload_file" to specify the filename.

Note

Workload dataset should be saved in .csv format and should contain data for one year (24*365), in a hourly way.

Example:

'workload_file': 'Alibaba_CPU_Data_Hourly_1.csv'

Agent configuration

DCRL-Green supports three MARL agents to optimize energy usage and reduce carbon footprint of data centers. More details of the agents’ operations are discussed in Agents section. Based on the requirement, users can include agents of choice in the training script. The agents that are not involved in training will select the Idle action by default and will not contribute to the optimization process. The variable "agents" can be used to specify the required agents.

Note

Agent names must be provided as a list of strings, where "agent_ls", "agent_dc", "agent_bat" represents load shifting agent, DC cooling agent, battery control agent respectively

Example:

'agents': ['agent_ls','agent_dc', 'agent_bat']
